autenticación de usuarios y funcionalidades de administración (rol admin) + mantenedor de proveedores consumiendo JSON.

3.4 Pantallas obligatorias + formularios con validaciones
Pantallas obligatorias solicitadas:
- Inicio de sesión: /login (Reactive Forms, validación required + email)
- Registro: /register (Reactive Forms, validación nombre/email/edad/password)
- Recuperar contraseña: /forgot (Reactive Forms, validación email)
- Modificación de perfil: /profile (Reactive Forms, validación nombre/email/edad, cambio de contraseña opcional)
Páginas internas según temática “ventas”:
- Home: /
- Productos: /products
- Detalle producto: /product/:id
- Carrito: /cart
- Mis compras: /orders
- Panel Admin: /admin
- Mantenedor Proveedores (Admin): /admin/proveedores

3.5 Roles / privilegios (mínimo 2 roles)
- Roles definidos: 'admin' y 'cliente'
- Rutas protegidas por guards:
  - /admin y /admin/proveedores requieren AuthGuard + RoleGuard (solo admin)
- En el navbar se muestra menú de “Panel administrador” solo cuando usuarioActual.rol === 'admin'
Credenciales de prueba (admin por defecto):
- Email: admin@tiendaderatas.cl
- Password: Admin123*
(El usuario “cliente” se crea con el Registro.)

3.6 Validaciones de seguridad de contraseña (mínimo 4)
Registro y Perfil aplican validaciones de password:
- required
- minLength(6)
- maxLength(18)
- pattern: al menos 1 mayúscula + 1 número
Sugerencia para cumplir al 100% con el ejemplo de “caracteres especiales”:
- Ajustar el pattern para exigir 1 caracter especial (por ejemplo: !@#$%^&* etc.)

3.8 Consumo y manipulación de JSON (backend “simulado”)
- La aplicación consume un JSON de proveedores:
  - Seed remoto (GitHub Pages): https://jagoldemberg-dotcom.github.io/proveedoreseed/proveedores.json
  - Fallback local: assets/data/proveedores.json
- La manipulación (CRUD) se realiza sobre LocalStorage como simulación (POST/PUT/DELETE).
- Funcionalidad extra: “Reset Seed” restaura la lista desde el JSON (útil para demostrar en video).

Nota / Sugerencia (si el docente exige que TODA la data venga de JSON):
